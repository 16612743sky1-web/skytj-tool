# 퓨어 베이비 츄잉 — 톤앤매너 유지 버전 1차 배포 계획

2026-07-17 · 대상: `petfocus-detail/index.html` · 사용자 승인: 대화에서 ①안 확정
("한글 얹기 편집기는 제외하고 먼저 배포, 한글 얹기는 당분간 미리캔버스 등 외부 도구 사용")

## 확정된 방향 (사용자 답변 반영)

- 기존 상세페이지(`상세페이지(원본)/`) 톤앤매너 유지 + 상품 부분만 교체
- 공통 정보(브랜드·지침·톤앤매너)는 1번만 입력, 상품마다 제품 정보만 입력
- **모든 제품 단품 생산·판매** (세트 없음), 용량 추가 생산 가능성 → 용량 단정 문구 금지
- 색감은 최종 결정자가 따로 있음 → 추천안(크림·베이지+민트)/대안(기존 하늘·옐로우) **재생성 가능한 선택 구조**
- 퍼피 실촬영본 확보 예정 → 촬영본 보정(1순위) 원칙 유지
- **상품 고유 정보를 코드에 하드코딩하지 않는다** (퓨어 베이비 츄잉 불러오기 버튼 1개만 예외) — 다음 신상품은 데이터 추가만으로 사용 가능해야 함

## 방법

### A. 3단계에 "기존 상세페이지 톤앤매너" 공통 필드
- `state.guidelines.tonemanner` textarea + 기본값(기존 상세페이지 분석 요약: Check Point 패턴, ✔ 체크, 급여량 표, ※ 각주, 다정한 존댓말, 여백·실사 중심)
- `load()`에서 필드 없으면 기본값 주입 (키 버전업 불필요 — 하위 호환 추가)
- `build3`·`build4`·`build5c`에 `[기존 상세페이지 톤앤매너 — 유지]` 블록 + "구성·말투는 유지하고 상품 관련 내용만 이번 제품으로 교체" 지시

### B. 상품별 색감 선택 + 상품 전용 섹션 구성 (선택)
- 상품 필드 `palette`: `mint`(크림·베이지·브라운+민트 — 기본·추천) / `sky`(기존 하늘·옐로우) — 5단계에서 선택, `build5b`/`build5c`에 반영. 바꾸고 다시 복사하면 재생성.
- 상품 필드 `sections`(textarea, 선택): 비우면 공통 지침 섹션, 채우면 이 상품만 그 구성 사용 — `build4`/`build5c` 적용

### C. 홈 "🐶 퓨어 베이비 츄잉 불러오기" 버튼
- 같은 이름 상품 있으면 선택만, 없으면 PDF 기획안 데이터로 생성:
  제품명·원재료(100% 국내산 한우 우피, 무첨가·무방부제·색소/향미제 무첨가)·규격(5~6cm 미니, 35g/200g 단품)·가격 빈칸·특징·급여·주의·사진 콘셉트·상품 전용 10섹션 구성
- 브랜드가 비어 있으면 펫포커스 기본값 저장 (공통 지침은 건드리지 않음)

### D. 지난 세션 완료분
- "📷 이미지 삽입" 기능 (검증 완료) 이번 커밋에 포함

## 리스크

- 기존 localStorage 데이터와의 호환: 새 필드는 전부 optional + load()/사용처에서 기본값 처리
- 시드 중복 클릭: 이름 중복 검사로 방지
- 한글 인코딩: UTF-8 유지, 배포 전 브라우저 렌더 확인

## 검증 (헤드리스 Chrome + 주입 하네스)

1. 첫 로드 콘솔 오류 0건
2. 시드 버튼 → 상품 생성, 1~6단계 프롬프트 전부 채워짐, 중복 클릭 시 중복 생성 없음
3. 4단계 프롬프트에 톤앤매너 블록 + "상품 부분만 교체" 지시 + 상품 전용 섹션 포함
4. palette 변경 시 5단계 프롬프트 색감 문구 변경
5. 톤앤매너 필드 없는 기존 저장 데이터 로드 → 오류 없이 기본값 주입
6. 이미지 삽입 회귀 (기존 하네스 10항목)
7. 새로고침 유지

검증 통과 후 deploy-check → 커밋 → push (GitHub Pages 배포)
