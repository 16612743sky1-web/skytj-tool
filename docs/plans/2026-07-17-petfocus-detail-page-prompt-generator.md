# 펫포커스 상세페이지 프롬프트 생성기 (계획서)

작성일: 2026-07-17
브랜치: `claude/product-detail-automation-lbc1d7`

## 한 줄 요약

상품 정보를 입력하면 **AI(미리클·젠스파크·ChatGPT 등)에 그대로 붙여넣을 프롬프트**를
6단계로 자동으로 뽑아주는 도구. 상세페이지 이미지를 직접 그리는 게 아니라,
**"AI에게 시킬 말"을 상품마다 자동 완성**해줘서 외주(20~80만원/개) 없이 사내에서 찍어낸다.

- 재고관리 도구와 **완전히 분리** — 새 폴더 `petfocus-detail/`, 별도 라이브 주소
- 정적 HTML 1개, 서버 없음, 데이터는 브라우저(localStorage) 저장
- 참고한 방식: "브랜드 정의 → 제품 정보 → 지침 → 텍스트 구성 먼저 → 이미지 프롬프트 → 제작" 6단계

---

## ★ 대표 최우선 요구사항 (실패 방지 2가지)

이 도구가 반드시 풀어야 하는 두 문제 — 안 풀리면 상세페이지 AI 자동화는 실패한다.

### 최우선 1: 한글 깨짐 방지 (AI 이미지의 고질병)
- AI(미리클·나노바나나·미드저니 등)는 **이미지 안에 한글을 그리면 깨진다**(외계어). 프롬프트로 못 막음.
- **확실한 해법 = 텍스트/이미지 분리**:
  1. 이미지는 **글자 없이**(배경·제품컷·무드컷만) 생성 → 프롬프트에 "이미지에 어떤 글자도 넣지 마" 기본 삽입
  2. 한글 카피는 도구에서 **따로 복사** → 미리캔버스/캔바/파워포인트에서 **진짜 한글 폰트로 글자 레이어**를 얹음
  - 결과: 한글 100% 안 깨짐. 이 워크플로를 도구가 강제하고 설명서로 안내.

### 최우선 2: "AI 느낌" 제거 (글 + 사진)
- **글의 AI 티**: 카피 프롬프트에 **탈-AI 문체 규칙** 기본 삽입
  · 금지 클리셰: "혁신적인·완벽한·최고의·특별한·지금 바로·경험해보세요" 등
  · 문장 길이 다양화, 이모지 남발 금지, 구체적 수치·감각(냄새·식감) 묘사, **등록한 브랜드 실제 말투 유지**
- **사진의 AI 티**: **촬영본 기반 보정(image-to-image)을 1순위**로 → 실물이라 AI 티 최소
  · 완전 생성(text-to-image)은 보조로만, "과하게 매끈/채도 높음 금지, 자연광·실사 질감" 지시 삽입

> 이 2가지는 5·6단계 프롬프트와 설명서 전반에 관통 반영한다.

## 목표

1. 신상품이 나올 때마다 **폼만 채우면** 상세페이지용 프롬프트가 6단계로 완성된다.
2. **공통 정보(브랜드·지침)는 한 번만** 저장 → 100개 상품이어도 재입력 안 함.
3. 상품별로 다른 것(이름·성분·특징·가격·사진 콘셉트)만 입력.
4. 각 단계 프롬프트는 **[복사] 버튼** 한 번으로 클립보드에 → AI 툴에 붙여넣기.
5. 사료/간식 **표시광고 규제**(효능 단정·과대광고)를 피하는 문구 규칙을 기본 내장.

## 도구 구성 (6단계 = 참고 글 그대로)

앱 상단에 상품 선택 바가 있고, 아래 6개 단계 탭을 오간다.
각 단계는 **입력 폼 + 자동 생성된 프롬프트 박스([복사] 버튼)** 로 구성.

### 0. 상품 관리 (홈)
- 저장된 상품 목록, `+ 새 상품` 버튼
- 상품 하나 = 이 6단계 데이터 한 세트

### 1. 브랜드 정보 (공통 · 한 번만 저장)
입력: 브랜드명(펫포커스) · 업종 · 브랜드 가치/톤 · 타깃 고객 · 경쟁사 대비 차별점
→ 출력: **"우리 브랜드를 정의/리서치해줘" 프롬프트**

### 2. 제품 정보 (상품별)
입력: 제품명 · 핵심 원재료(한우 우피 등) · 규격/중량/개수 · 특징 3~5개 · 급여법 · 가격 · 주의사항
→ 출력: **"이 제품 정보를 상세페이지용으로 정리해줘" 프롬프트**

### 3. 상세페이지 지침 (공통 · 규칙)
입력(기본값 제공): 톤앤매너 · 필수 섹션 순서 · **금지 표현**(질병 치료·예방 단정, 최고/유일 등 과장)
- **신생 브랜드용 신뢰 우선 구성**(기본값): 후킹 → 문제제기 → 해결 → **성분 근거/출처** →
  **신뢰요소(수제·소량생산·원산지·무첨가 안심포인트)** → 급여법 → 주의사항 → 브랜드 스토리(왜 만들었나) → FAQ/배송
  · 이유: 아직 인지도가 낮으므로 상세페이지가 신뢰를 직접 쌓아야 함(#2)
- **기존 톤 등록 필드**: 현재 상세페이지 톤/문장을 붙여넣거나 요약 입력 → 새 상품 프롬프트가 그 톤을 유지(#3)
  · 신생 브랜드일수록 페이지 간 톤 일관성이 브랜드 인지에 중요 → 한 번 등록해 100개에 재사용
→ 출력: **"상세페이지가 지켜야 할 규칙 + 이 톤을 유지" 프롬프트**

### 4. 텍스트 구성 먼저 확정
- 1~3단계를 합쳐 **"디자인 전에 텍스트로 전체 구성/카피부터 짜줘"** 통합 프롬프트 자동 생성
- (참고 글 핵심: 디자인 전에 글부터 확정해야 재작업이 없다)
→ 출력: 붙여넣으면 AI가 섹션별 헤드라인·본문을 뽑아주는 프롬프트

### 5. 이미지 프롬프트 (전문장비 없는 상황 대응 · #5)
촬영본만 있고 스튜디오 장비가 없으므로 **두 갈래**로 나눔:
- 5-A) **촬영본 살리기 프롬프트** — 가진 휴대폰 촬영본을 AI로 배경 정리·밝기/색보정·
  스튜디오 느낌 보정하도록 지시하는 프롬프트 (누끼/배경 교체 포함)
- 5-B) **AI 실사 생성 프롬프트** — 직접 못 찍는 컷(급여 장면·성분 클로즈업·브랜드 무드컷)을
  새로 생성하는 프롬프트 (섹션별, 400자 이내)
- 5-C) **상세페이지 생성 프롬프트** — 미리클 등에 넣어 최종 이미지로 조합
- **모든 이미지 프롬프트에 "글자 없이 생성" 문구 자동 삽입**(한글 깨짐 방지, 최우선 1)
- 카피(4단계 텍스트)는 **별도 [복사]** → 미리캔버스/캔바에서 한글 폰트로 얹음
→ 출력: 이미지 툴용 프롬프트 3종 + 얹을 한글 카피(분리)

### 6. 제작 · 수정 가이드
- 미리클/젠스파크/ChatGPT에서 **붙여넣는 순서 체크리스트**
  (촬영본 보정/실사 생성 → 첨부 → 상세페이지 생성 → **미리캔버스/캔바에서 한글 카피 얹기**)
- **한글 얹는 법 단계별 안내**(폰트 추천·글자 크기·정렬) — 이미지엔 글자 없이, 텍스트는 위에 덮기
- **수정 요청 프롬프트 도우미**(#4 핵심): "이 부분을 이렇게 바꿔줘" 템플릿 제공 →
  외주는 수정마다 비용이지만 여기선 **무료·무제한 재생성**. 수정=돈 문제 해소.
- 실제 이미지 생성은 그 AI 툴에서 (이 도구는 프롬프트까지 담당)

## 사용자(사장님)가 준비할 것 — "무엇부터 하나씩"

도구가 완성되면 아래 순서대로 하면 됩니다. **딱 한 번만 하는 것**과 **상품마다 하는 것**을 구분했어요.

**A. 딱 한 번만 (공통, 처음 세팅)**
1. 브랜드 한 줄 소개 — 펫포커스가 어떤 브랜드인지 (예: "한우 우피로 만든 프리미엄 수제 강아지 간식")
2. 우리 브랜드의 강점 2~3개 (예: 국내산 한우 우피, 수제 소량생산, 무첨가)
3. 주 고객 (예: 소형견 보호자, 치석·구취 신경 쓰는 분)
4. 쓸 AI 툴 정하기 (미리클 / 젠스파크 / ChatGPT 이미지 등 — 무료로 시작 가능)

**B. 상품 하나 나올 때마다**
1. 제품명
2. 핵심 원재료·성분
3. 규격(중량·개수)·가격
4. 특징/장점 3~5개 (한 줄씩)
5. 급여 방법 + 주의사항
6. (선택) 넣고 싶은 사진 콘셉트 (예: 강아지가 우피껌 물고 있는 컷)

**C. 도구가 만들어준 프롬프트로**
1. 1~4단계 [복사] → AI에 붙여넣어 **텍스트 구성부터** 확정
2. 5단계 [복사] → 실사 이미지 프롬프트로 사진 생성
3. 6단계 체크리스트대로 → 최종 상세페이지 이미지 생성 → 쿠팡 업로드

## 리스크 / 주의

- **표시광고 규제**: 간식이라도 "치석 제거·질병 예방" 같은 효능 단정은 제재 위험 → 3단계 금지표현에 기본 반영, 생성 프롬프트에 "단정 표현 금지" 문구 포함.
- 이 도구는 **프롬프트까지** 담당. 실제 이미지 품질은 사용하는 AI 툴에 좌우됨(도구 밖 영역).
- 상품 데이터가 많아지면 localStorage 용량(브라우저별 5~10MB) 고려 — 텍스트만 저장하므로 100개도 문제없음(사진 원본은 저장 안 함).
- 동기화(재고관리처럼 Firebase)는 이번 범위에서 **제외**. 우선 로컬 저장으로 시작, 필요하면 다음에 추가.
