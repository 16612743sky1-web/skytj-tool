# skytj-tool

핸드폰/PC 브라우저에서 바로 쓰는 **회사업무(카드밴대리점) 도구 모음**.
서버 없이 정적 HTML로만 구성되며 GitHub Pages로 배포한다.

> 📌 2026-07-19 정리: 이 저장소는 **회사업무 전용**이다.
> 예전에 섞여 있던 펫포커스 도구(재고/원가·상세페이지 생성기)와 마케팅 문서는
> 별도 저장소 **`petfocus-inventory`** 로 이관했다.
> `petfocus/`·`petfocus-detail/` 에 남은 것은 옛 주소를 새 주소로 넘겨주는 **리다이렉트 페이지**뿐이니,
> 펫포커스 관련 작업은 이 저장소가 아니라 `petfocus-inventory` 에서 한다.

## 작업 방식 (중요)

- **코드를 고치기 전에 계획서부터 쓴다.** 도구(HTML)의 동작을 바꾸는 작업은 바로 시작하지 말고,
  먼저 관련 파일을 파악한 뒤 `docs/plans/`에 계획서(목표·바꿀 파일·방법·리스크·검증)를 작성하고
  핵심을 요약해 보여준다. 그 다음 **사용자 승인을 받고 나서** 실행한다.
  (`/plan` 명령어와 동일한 절차 — 사용자가 명령어를 치지 않아도 기본 적용)
- 승인 전에는 코드를 수정하지 않는다. 사용자가 방향을 바로잡으면 계획서를 고친 뒤 다시 승인을 기다린다.
- 단, 오타·문구·문서 수정 같은 사소한 변경은 계획서를 생략하고 바로 해도 된다. 애매하면 계획서를 쓰는 쪽.
- 작업 후에는 아래 "검증" 기준으로 점검하고, 문제가 있으면 배포하지 않고 먼저 보고한다.

## 구성

| 경로 | 도구 | 사용자 |
|---|---|---|
| `index.html` | 스카이티제이 월말 정산 — 에이스아이피 발송 파일과 자체 택배 기록(엑셀)을 대사하여 불일치·KIS 키핑재고 차감·최종 정산액 산출 | 스카이티제이 (카드밴대리점) |
| `claude-usage/index.html` | 클로드 사용량 대시보드 — Claude 앱/웹 설정의 "플랜 사용량 한도"(현재 세션 + 주간 한도: 모든 모델·Opus·Fable 등) 값을 직접 입력하면 링 게이지 + 재설정 카운트다운으로 시각화, PNG 저장 (공식 API 없어 수동 입력). **개인용** — 회사 PC에서 사용량을 띄워 보려는 용도 | 개인 |
| `petfocus/index.html` | (리다이렉트) → `petfocus-inventory/stock-cost/` 로 이동 | — |
| `petfocus-detail/index.html` | (리다이렉트) → `petfocus-inventory/detail/` 로 이동 | — |

## 개발 원칙

- **도구 1개 = HTML 파일 1개.** 빌드 도구·프레임워크·npm 의존성 금지. CDN 스크립트는 xlsx 등 최소한만.
- 모바일 우선 (390px 기준), 한국어 UI, Pretendard 폰트, 기존 도구와 동일한 CSS 변수 팔레트 유지.
- 데이터는 `localStorage` 저장 (서버 없음).
- `petfocus/`·`petfocus-detail/` 의 리다이렉트 stub 은 옛 북마크 보호용이다. 지우거나 앱을
  되돌려 넣지 말 것 — 펫포커스 실제 도구는 `petfocus-inventory` 저장소에서 관리한다.

## 검증

자동 테스트 없음. 변경 시 Playwright(`executablePath: '/opt/pw-browsers/chromium'`)로 파일을 직접 열어
핵심 흐름을 확인하고 콘솔 오류를 점검한다.
- `index.html`(월말정산): 에이스아이피 파일 + 택배 기록 업로드 → 대사 결과·불일치·KIS 차감·정산액 표시.
- `claude-usage/`: 사용량 입력 → 링 게이지·카운트다운 → PNG 저장.
- 리다이렉트 stub: 새 주소로 실제 이동하는지 확인.
